# CONY 프로젝트 포트폴리오 정리

## 1. 프로젝트 개요

### 프로젝트 한 줄 소개
**CONY**는 기프티콘을 등록/관리하고, 위치 기반 알림(Geofence)과 거래/결제, OCR/AI 추천까지 제공하는 **모바일 중심 기프티콘 통합 관리 서비스**입니다.

### 프로젝트 목표와 문제 정의
- 사용자가 보유한 기프티콘을 체계적으로 관리하기 어렵고, 유효기간 만료/미사용으로 인한 손실이 자주 발생
- 기프티콘 활용도를 높이기 위해 **근처 매장 도착 시점에 즉시 사용 가능한 쿠폰을 안내**하는 경험 필요
- 단순 보관을 넘어, 만료 임박 쿠폰의 거래/추천/자동 판매까지 연결하는 서비스 필요

### 핵심 기능
- **기프티콘 관리**: OCR 기반 자동 등록, 직접 등록, 조회/수정/삭제, 사용/사용취소, 만료 관리
- **위치 기반 알림**: Geofence 진입 시 보유 쿠폰과 매장 매칭 후 푸시 알림 발송
- **거래/결제**: 판매/구매/거래내역, 카카오페이 연동 포인트 결제
- **추천**: 거리/유효기간 기반 일반 추천 + 사용자 로그 기반 AI 개인화 추천
- **알림 체계**: FCM 푸시 + 알림 히스토리 저장 + 스케줄러 기반 만료 임박 알림

### 시스템 구조
- **Client**: React Native 앱(`CONY-APP`), Next.js 웹(`CONY-WEB`, 코드 잔존)
- **Backend (MSA)**
  - `manage`: 인증, 기프티콘, 공유방, Geofence, 알림
  - `payment`: 포인트/결제, 판매/구매/추천
  - `ai`: OCR/임베딩/추천(FastAPI)
- **Data/Infra**: MySQL, Redis, AWS S3, Docker Compose, Jenkins, Prometheus/Grafana

### 기술 스택

| 구분 | 기술 |
|---|---|
| Mobile | React Native, TypeScript |
| Backend | Java 17, Spring Boot, Spring Security, JPA, Spring Batch |
| AI | Python, FastAPI |
| DB/Cache | MySQL, Redis |
| Infra | Docker, Jenkins, Nginx, AWS S3, Prometheus, Grafana |
| External | Google/Kakao OAuth, KakaoPay, Firebase Cloud Messaging |

### 프로젝트 특장점
- 모바일 네이티브 기능(Geofence/Widget)과 MSA 백엔드, 외부 결제/알림/AI 연동을 결합한 **실전형 서비스 아키텍처**
- Redis GEO + 배치 캐시 + 클러스터링 최적화로 위치 기반 기능의 성능/운영 안정성을 확보
- 결제/거래 도메인과 기프티콘 도메인의 상태 정합성을 내부 API로 명확히 관리

## 2. 프로젝트에서 담당한 파트

## 담당 역할 요약 (정혜원)
- **백엔드 핵심 개발자**로서 Manage/Payment 중심 도메인 설계 및 구현
- Geofence, 알림, 추천, 기프티콘 관리, 내부 API 보안/정합성 등 핵심 기능 전반 담당
- 커밋 기준 기여: `정혜원` + `Hyewon_Jung` 합산 **244건** (전체 918건 중 약 26.6%)

## 상세 담당 영역

### 1) 백엔드 공통 기반 구축
- 공통 응답 포맷(`ApiResponse`), 글로벌 예외 처리(`CustomException`, `ErrorCode`) 체계 설계
- 보안/Redis/JPA Auditing 등 서비스 공통 설정 기반 마련

### 2) 기프티콘 도메인 핵심 기능 구현
- Gifticon/Brand/Category/Image/UsageLog 등 핵심 엔티티 및 서비스 레이어 구축
- 동적 검색(브랜드/상품명/카테고리/만료임박/위치기반)과 정렬 고도화
- 논리 삭제(soft delete), 사용/사용취소, 이력 관리 로직 안정화

### 3) OCR 연동 파이프라인 구축
- OCR 서버 연동 API 및 DTO 매핑 구현
- 연동 포맷 이슈(snake_case) 해결로 등록 자동화 안정성 확보

### 4) Geofence + Redis + Batch 기반 위치 기능 설계
- Redis GEO/Hash 기반 매장 캐시 구조 설계
- Spring Batch를 통한 매장 데이터 워밍업 파이프라인 구현
- 사용자 위치 기준 매장 탐색, 보유 브랜드 필터링, 클러스터링(플랫폼 제약 대응) 로직 개발

### 5) 알림 시스템(FCM + 스케줄러) 구현
- Geofence 진입 알림, 만료 임박 알림, 자동판매 알림 연계
- 푸시 발송 실패 시에도 알림 이력을 저장하는 운영 친화적 구조 적용
- 정기 스케줄링(매일 오전 10시) 기반 만료 임박 알림 자동화

### 6) 추천 시스템 구현 (일반 + AI 개인화)
- 거리/유효기간 기반 일반 추천 로직 구현
- 사용자 상호작용 로그 기반 AI 추천 연동 및 fallback 전략 적용
- 추천 결과에 판매 정보/기프티콘 상세 정보를 병합하여 응답 완성

### 7) 서버 간 정합성/보안 강화
- `payment`-`manage` 간 상태 동기화(판매 시작/취소, 구매 후 소유권 이전) 구현
- 내부 통신 보호를 위한 `X-Internal-Key` 인터셉터 기반 인증 경계 도입
- 내부 API 통신 테스트 코드 작성으로 회귀 리스크 완화

## 기술적 특장점
- **도메인 정합성 설계**: 판매/구매/소유권 이전 시 상태 불일치 방지를 위한 내부 API 분리 및 예외 처리
- **성능 최적화**: Redis GEO + Pipeline + Batch로 위치 조회의 응답성 개선
- **운영 관점 개발**: 알림 실패 시 이력 보존, 배치/스케줄 기반 자동화, 테스트 보강
- **보안 강화**: 내부 API 키 인증 경계 추가 및 민감정보 정리 대응

## 리스크 관리 방식
- **정합성 리스크**: MSA 간 상태 동기화 실패 시 도메인 예외 처리로 불완전 트랜잭션 방지
- **성능 리스크**: DB 직접 탐색 대신 캐시 중심 구조로 위치 기반 트래픽 대응
- **알림 품질 리스크**: 브랜드 단위 쿨다운 정책으로 중복 알림 피로도 완화
- **연동 장애 리스크**: 외부 API 실패 시 핵심 데이터(알림 이력) 보존 우선
- **보안 리스크**: 내부 호출 인증 강제 및 민감정보 제거 커밋으로 반출/운영 리스크 축소

## 포트폴리오 기재용 요약 문구

CONY 프로젝트에서 백엔드 핵심 개발자로 참여하여, 기프티콘 관리 도메인과 Geofence 기반 알림 기능을 중심으로 결제/추천 시스템까지 확장했습니다. Redis GEO + Batch 기반 최적화로 위치 기능의 성능을 개선했고, MSA 간 내부 API 보안 및 상태 동기화 설계를 통해 운영 안정성과 데이터 정합성을 확보했습니다.
